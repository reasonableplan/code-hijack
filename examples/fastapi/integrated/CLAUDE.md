## Top MUST Rules (All Layers)

*6 most critical rules across all categories:*

- **[backend/MUST]** Subclass Starlette/Pydantic primitives instead of wrapping them; expose FastAPI as a thin typed layer on top of lower-level ASGI framework.
- **[backend/MUST]** Keep exception type hierarchy distinct by audience: HTTPException for client errors, FastAPIError (RuntimeError) for developer/framework misuse, plus separate WebSocket and Validation branches.
- **[backend/MUST]** All public constructor/function parameters past `self` must be keyword-only (use `*,` separator); do not accept positional arguments for configuration options.
- **[backend/MUST]** Every authentication dependency must accept `auto_error: bool = True` and honor it: raise HTTPException(401) when True, return None when False.
- **[backend/MUST]** HTTP 401 responses must always attach a WWW-Authenticate header with the correct challenge scheme (Bearer / Basic / APIKey), not just a JSON detail.
- **[backend/MUST]** Raise HTTPException with a specific status code and short human-readable `detail`; do not return a dict with an error key or a 200 response carrying error semantics.
